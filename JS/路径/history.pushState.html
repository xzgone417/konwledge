`history.pushState(state, title, url)` 和 `history.replaceState(state, title, url)` 是 HTML5 提供的用于操作浏览器历史记录的 API。它们允许在不刷新页面的情况下更改浏览器的 URL，并添加或替换历史记录条目。这两者的区别和特性如下： ### 1. `history.pushState(state, title, url)` - **作用**：在浏览器的历史记录中添加一个新条目，并将 URL 改为指定的值，同时保留当前页面的状态。 - **参数**： - `state`：可以是任意可序列化的对象，通常用于保存当前页面的状态。在页面状态发生变化时，这个对象可以用来记录状态，并且在用户通过浏览器的前进或后退按钮返回到这个状态时可以被恢复。 - `title`：当前页面的标题，但现代浏览器通常忽略这个参数，所以可以传递 `null` 或空字符串。 - `url`：新的 URL（必须与当前域名相同）。这个 URL 会显示在浏览器地址栏中，并成为新的历史记录条目。如果 `url` 是 `null`，则不会改变当前的 URL。 - **特性**： - 该方法不会触发页面刷新。 - 不会自动发出 HTTP 请求，即只是在客户端修改 URL。 - 在调用该方法后，浏览器的“后退”按钮会将用户带回到之前的页面状态。 ### 2. `history.replaceState(state, title, url)` - **作用**：与 `pushState` 类似，但它不会添加一个新的历史记录条目，而是替换当前的历史记录条目。 - **参数**： - `state`：与 `pushState` 的 `state` 参数相同，用于保存当前页面的状态。 - `title`：与 `pushState` 的 `title` 参数相同，通常也被忽略。 - `url`：新的 URL，用于替换当前历史记录条目中的 URL。 - **特性**： - 不会触发页面刷新。 - 替换当前的历史记录条目，而不是创建一个新的。 - 适用于需要更新 URL 或状态但不想影响历史记录堆栈的情况。 ### 默认值 - `state`：如果未提供 `state`，则通常为 `null`。 - `title`：通常浏览器不使用该参数，可以设置为 `null` 或空字符串。 - `url`：如果不提供 `url` 或者设置为 `null`，URL 保持不变。 ### 使用场景 - **pushState**：适用于页面内容发生较大变化时，需要为用户提供可以通过“后退”按钮返回到之前状态的功能。 - **replaceState**：适用于页面内容或状态变化不大，不希望用户可以通过“后退”按钮回到该状态，而仅仅需要更新 URL 或状态。 通过这两个 API，你可以灵活地管理单页面应用（SPA）的历史记录和浏览体验。
